# Техническое задание: одностраничный сайт для Barbershop Макса Елизаренко

## Общая информация

Необходимо разработать современный одностраничный сайт (Landing Page) для индивидуального Barbershop.

Владелец бизнеса и единственный мастер:

Макс Елизаренко

Он самостоятельно ведет весь бизнес, принимает клиентов и выполняет все услуги.

Сайт должен создавать ощущение персонального бренда мастера, доверия и премиального индивидуального обслуживания.

Основная задача сайта:

- представить мастера;
- показать полный список услуг и цены;
- дать клиенту возможность быстро связаться;
- предоставить удобную запись;
- показать отзывы клиентов;
- указать расположение Barbershop на карте;
- создать современный образ персонального Barbershop.

---

Использовать:

# Общая структура сайта

Сайт должен быть выполнен как вертикальный одностраничный Landing Page.

Пользователь должен прокручивать страницу сверху вниз.

Основные секции:

1. Главный экран / Обо мне
2. Услуги и цены
3. Отзывы клиентов
4. Контакты
5. Форма записи

# Верхняя навигация (Header)

В верхней части страницы должен находиться фиксированный Header.

Header должен содержать:

- название Barbershop;
- навигационные кнопки.

Навигация должна работать через плавную прокрутку к нужным секциям страницы.

Кнопки:

| Кнопка | Секция |
|---|---|
| Обо мне | Блок с информацией о мастере |
| Услуги | Блок услуг и цен |
| Отзывы | Блок отзывов |
| Контакты | Блок контактов |
| Запись | Форма записи |

Использовать плавный scroll behavior.

# Адаптивность и мобильная версия сайта

Сайт обязательно должен быть полностью адаптивным и корректно отображаться на мобильных устройствах.

Основной сценарий использования сайта — просмотр с телефона, поэтому мобильная версия должна быть полноценной, а не просто уменьшенной копией desktop-версии.

Требования:

- корректное отображение на смартфонах с разными размерами экранов;
- адаптация под вертикальную ориентацию мобильного устройства;
- удобная навигация на маленьких экранах;
- все кнопки должны иметь комфортный размер для нажатия пальцем;
- текст должен оставаться читаемым без увеличения страницы;
- изображения должны автоматически подстраиваться под размер экрана;
- карточки услуг должны перестраиваться в одну колонку;
- отзывы должны адаптироваться под мобильный формат;
- форма записи должна быть удобной для заполнения с телефона;
- контакты, WhatsApp, Telegram и Яндекс Карты должны быть легко доступны с мобильного устройства.

Header на мобильной версии:

- должен адаптироваться под маленький экран;
- навигация может быть реализована через мобильное меню (hamburger menu);
- переходы между секциями должны сохранять плавную прокрутку.

Использовать responsive CSS подход:

- CSS media queries;
- гибкие размеры блоков;
- адаптивные изображения;
- mobile-first подход.

Необходимо проверить отображение сайта:

- Desktop;
- Tablet;
- Mobile устройства iOS;
- Mobile устройства Android.

---

# GitHub Actions

Создать workflow:

.github/workflows/deploy.yml

Workflow должен:

1. Запускаться при push в основную ветку:

main

2. Устанавливать зависимости:

npm install

3. Выполнять сборку проекта:

npm run build

4. Получать итоговые файлы сборки:

dist/

или

build/

в зависимости от используемого React-сборщика.

5. Автоматически публиковать результат в ветку:

gh-pages
